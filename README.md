# gemini-canvas-mission

Gemini Canvas 미션 결과물을 관리하는 저장소입니다.

## 목적

- Gemini Canvas로 제작한 웹앱 결과물을 기록하고 공유합니다.
- 앱별 요구사항, 배포 링크, 회고를 일관된 형식으로 남깁니다.

* 반응형 모달 상세 보기: 선택한 레시피의 상세 정보를 깔끔한 레이아웃의 팝업창으로 확인할 수 있어 보기 편합니다.

### AI 기능
* Gemini AI 레시피 생성: 단순히 데이터베이스에서 검색하는 것이 아니라, Gemini AI 2.5 Flash 모델을 연동하여 사용자가 입력한 재료 조합을 분석하여 사용자의 재료에 맞춰 실시간으로 가능한 최적의 요리를 추천합니다.

## 카테고리: 페어 프롬프트 릴레이

### 배포 링크

### 이 앱을 만든 이유
- 코딩 테스트 준비 시 모든 문제를 직접 구현하는 방식은 물리적 시간 소모가 커서 다양한 유형을 섭렵하기 어렵다는 한계가 있습니다. 코드를 구현하는 것도 중요하지만 문제의 제약 조건을 분석해 최적의 자료구조와 알고리즘을 설계하는 안목이 가장 중요하다고 생각했습니다.
- 활용 방안: 이 앱은 구현 과정을 생략하고 '문제 해석 → 제약 조건 분석 → 최적 자료구조 및 알고리즘 선택'이라는 핵심 전략 수립 단계에만 집중하게 함으로써, 짧은 시간 동안 폭넓은 유형의 문제를 압축적으로 학습할 수 있도록 돕습니다.

### 주요 기능
* 난이도별 맞춤 문제 탐색: 초급, 중급, 고급으로 사용자가 원하는 수준에 맞는 문제를 실시간으로 불러옵니다.
* 실전 사이트 연동: 백준(BOJ), 프로그래머스 등 실제 코딩 테스트 플랫폼의 실존 문제와 원문 링크를 제공합니다.
* 알고리즘 전략 퀴즈: 문제의 제약 사항(N의 범위 등)을 보고 여러가지 후보 중 최적의 알고리즘 기법을 선택하는 인터랙티브 퀴즈를 제공합니다.

### AI 기능
* 실시간 문제 큐레이션 (Gemini 2.5 Flash): 구글 검색 기능을 활용하여 데이터베이스에 고정된 문제가 아닌, 웹상의 최신 기출 문제를 실시간으로 검색하고 난이도 기준에 맞춰 추출합니다.
* 심층 전략 분석 리포트: 사용자가 선택한 답변의 정답 여부뿐만 아니라, 시간 복잡도 계산 근거, 단계별 알고리즘 설계 프로세스, 핵심 마스터리 개념을 AI가 즉석에서 생성하여 심도 있는 피드백을 제공합니다.
* 중복 방지 로직: 사용자가 이미 풀었던 문제 제목을 기억하여 새로운 문제를 우선적으로 제시하도록 지능적으로 필터링합니다.

### 2. 프롬프트 개선 과정

### 3. 효과적이었던 프롬프팅 전략

- 어떤 방식으로 요청했을 때 원하는 결과가 나왔나요?

1. 단순히 "초급, 중급"이라고 하기보다 '백준 실버2,1 골드5,4,3' 처럼 구체적인 티어와 레벨을 명시했을 때 AI가 정확한 난이도의 문제를 가져왔습니다. 기준을 세분화할수록 결과의 정확도가 높아졌습니다.
2. "상세히 알려줘"라는 요청 대신 구조를 지정했을 때 사용자가 읽기 편한 결과가 생성되었습니다. 또한 UI적으로도 가독성이 확보되었습니다.
3. 수식 깨짐이나 마크다운 렌더링 문제를 해결하기 위해 'MathJax 문법 적용', '마크다운 렌더링 결과로 보여주기' 등 구체적인 기술 스택과 처리 방식을 지시했을 때 전문적인 웹 앱의 결과물이 나왔습니다.
4. ‘시뮬레이션 문제 제외', '중복 문제 제외', '함수 고르기/입출력 값 맞히기 유형 제외' 등 원치 않는 요소을 명확히 짚어서 제외 요청을 했을 때 더욱 원하는 결과가 나왔습니다.

- 반대로 효과가 없었던 방식은 어떤게 있었나요?

단순히 "문제 원문을 보여달라"고 했을 때는 개행 문자(\n)가 그대로 노출되거나 수학 기호가 깨지는 현상이 발생했습니다. MathJax 도입과 이스케이프 문자 처리를 명시하기 전까지는 렌더링 오류를 피할 수 없었습니다.

### 4. 배운 점

- Gemini Canvas를 효과적으로 사용하기 위한 나만의 팁이 있다면?

요청 기준을 세분화하고, 상세한 구조를 지정하고, 구체적인 추가 조건을 포함하여 지시한다면 더욱 원하는 결과물을 얻을 수 있습니다.

1. 초급/중급/고급 난이도에 따른 체감 난이도가 적절하다고 느껴지는지 궁금합니다.
2. 해설 리포트의 내용이 학습을 하는데 충분한 설명으로 느껴지는지 궁금합니다.
3. 작은 모바일 화면에서도 문제가 잘 읽혀질만큼 UI가 충분히 쾌적하게 느껴지는지 궁금합니다.
